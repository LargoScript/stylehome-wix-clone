# Інструкція з налаштування GitHub Pages

## Крок 1: Увімкнути GitHub Pages в налаштуваннях репозиторію

1. Перейдіть на GitHub репозиторій: `https://github.com/LargoScript/stylehome-wix-clone`
2. Натисніть на **Settings** (Налаштування)
3. У лівому меню знайдіть секцію **Pages**
4. У розділі **Source** виберіть:
   - **Source**: `GitHub Actions` (не "Deploy from a branch")
5. Збережіть зміни

## Крок 2: Перевірити, чи workflow запускається

1. Перейдіть на вкладку **Actions** у вашому репозиторії
2. Перевірте, чи є workflow "Deploy to GitHub Pages"
3. Якщо workflow не запускається автоматично, натисніть на нього та виберіть **Run workflow**

## Крок 3: Перевірити статус деплою

1. Після успішного виконання workflow перейдіть на вкладку **Environments**
2. Там має бути environment `github-pages` з URL вашого сайту
3. URL зазвичай виглядає так: `https://largoscript.github.io/stylehome-wix-clone/`

## Можливі проблеми та рішення

### Проблема: Workflow не запускається
**Рішення**: Переконайтеся, що:
- Файл `.github/workflows/deploy.yml` знаходиться в гілці `main`
- GitHub Pages налаштований на використання GitHub Actions

### Проблема: Build падає з помилкою
**Рішення**: 
- Перевірте логи в Actions
- Переконайтеся, що всі залежності встановлені (`npm ci` має пройти успішно)
- Перевірте, чи TypeScript компілюється без помилок

### Проблема: Сайт не відображається правильно
**Рішення**:
- Перевірте, чи base path правильний: `/stylehome-wix-clone/`
- Переконайтеся, що файл `.nojekyll` створюється в `dist/`
- Перевірте, чи всі шляхи до ресурсів правильні

## Ручний запуск деплою

Якщо потрібно запустити деплой вручну:

1. Перейдіть на вкладку **Actions**
2. Виберіть workflow "Deploy to GitHub Pages"
3. Натисніть **Run workflow**
4. Виберіть гілку `main`
5. Натисніть **Run workflow**

## Перевірка локальної збірки

Перед деплоєм можна перевірити збірку локально:

```bash
npm run build
npm run preview
```

Це запустить preview версію з правильним base path.
